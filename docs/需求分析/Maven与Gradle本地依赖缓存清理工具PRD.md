## 1. 产品概述

本产品是一款面向 Java 开发者的开源桌面客户端工具，用于扫描并安全清理本机：

- Maven 本地仓库
- Gradle 本地缓存

其核心目标不是“暴力删文件”，而是提供一套：

- 可视化扫描
- 空间占用分析
- 旧版本识别
- 可清理建议
- 风险提示
- 安全删除

的一站式本地依赖治理能力。

产品首发以 **桌面 GUI 客户端** 为主，方便普通开发者直接下载安装使用；后续可视情况扩展 CLI。

---

## 2. 产品背景

Java 开发者在长期使用 Maven / Gradle 过程中，本地依赖缓存会持续膨胀：

- 同一个 artifact 存在多个历史版本
- Gradle caches 中存在无效缓存、旧 wrapper、transforms 残留
- 本地仓库占用空间大，但难以直观看清来源
- 手动删除风险高，不知道哪些版本仍被项目使用

虽然开发者可以手动进入 `.m2` 或 `.gradle` 目录删除文件，但存在明显问题：

- 不知道删哪些最安全
- 不知道删完是否影响当前项目
- 不知道预期能释放多少空间
- 不知道 Maven / Gradle 是否还能正常构建

因此需要一款面向开发者的桌面客户端，帮助他们更安全、更可视化地完成本地依赖缓存治理。

## 5. 用户痛点

### 5.1 看不见

用户知道 `.m2` 和 `.gradle` 很大，但不知道：

- 到底哪个依赖占得最多
- 哪些 artifact 有最多历史版本
- 哪些 Gradle 缓存最占空间

### 5.2 不敢删

用户担心：

- 删掉后影响当前项目
- 私服或外网不可访问时无法重新下载
- 删掉 wrapper 或缓存后构建失败

### 5.3 手动处理成本高

用户需要自己：

- 进入隐藏目录
- 理解目录结构
- 逐个版本判断
- 手动删除

体验差且效率低。

### 5.4 缺少统一工具

现有磁盘清理工具通常只按文件大小删文件，不理解：

- Maven 坐标
- Gradle cache 结构
- 项目引用关系
- Java 开发场景风险

## 8. 使用场景

### 场景 1：清理长期积累的 Maven 旧版本

用户安装应用后，扫描 `.m2/repository`，发现某些 artifact 有十几个版本，只保留最近 2 个版本，删除其余版本。

### 场景 2：清理 Gradle 无效缓存

用户扫描 `.gradle/caches`，找到长期未使用的缓存目录、旧 wrapper 和异常缓存，执行清理并释放磁盘空间。

### 场景 3：项目切换后做一次安全整理

用户在多个项目之间切换数月后，使用本工具统一扫描本地依赖，查看哪些版本已经长期未使用，再手动确认删除。

### 场景 4：磁盘告急时快速释放空间

用户发现系统磁盘空间不足，希望快速查看 Maven / Gradle 缓存中“体积最大、风险较低”的可清理项。

## 10. 功能范围

### 10.1 V1 核心能力

#### 功能 1：本地路径发现

自动发现：

- Maven 本地仓库路径
- Gradle caches 路径
- Gradle wrapper 路径

并支持用户手动修改或新增扫描路径。

#### 功能 2：缓存扫描

扫描并展示：

- 总占用空间
- Maven / Gradle 各自占用空间
- 最大目录排行
- 重复版本排行
- 旧版本分布
- Gradle wrapper 分布

#### 功能 3：可清理项识别

识别以下候选项：

- 同一 artifact 的旧版本
- 180 天未使用版本
- `.lastUpdated`
- 下载失败残留
- 过期 SNAPSHOT
- Gradle 无效缓存
- 旧版 wrapper 压缩包与 distribution

#### 功能 4：清理建议

提供规则化建议，例如：

- 每个 artifact 保留最新 2 个版本
- 删除长期未使用缓存
- 仅清理低风险项

#### 功能 5：清理预演

在真正删除前，展示：

- 将删除哪些内容
- 预计释放多少空间
- 风险提示
- 当前保护规则

#### 功能 6：执行删除

支持：

- 勾选删除
- 一键按建议删除
- 删除结果反馈
- 失败项提示

#### 功能 7：安全保护

支持：

- 删除前确认
- 白名单
- 项目引用保护（V1 可做基础版）
- 尽量移入回收站 / 废纸篓
- 删除失败回显原因

## 11. 详细功能需求

### 11.1 首页 / 总览页

#### 目标

让用户第一眼看到本机 Maven / Gradle 缓存占用情况。

#### 展示内容

- 总缓存大小
- Maven 占用
- Gradle 占用
- 可清理空间预估
- 最近一次扫描时间
- “开始扫描”按钮

### 11.2 扫描结果页

#### 展示维度

- 按工具分类：Maven / Gradle
- 按 artifact 聚合
- 按版本聚合
- 按空间大小排序
- 按最后访问时间排序
- 按可清理优先级排序

#### 交互能力

- 搜索
- 筛选
- 展开某依赖查看版本
- 查看文件路径
- 加入白名单

### 11.3 清理建议页

#### 展示内容

- 推荐删除项列表
- 推荐原因
- 风险等级
- 预计释放空间

#### 风险等级建议

- 低风险：失败缓存、`.lastUpdated`、临时文件
- 中风险：旧版本但近期未使用
- 高风险：可能被项目或 wrapper 使用的版本

### 11.4 执行确认弹窗

删除前应明确展示：

- 删除项数量
- 总释放空间
- 风险提示
- 是否移动到回收站
- 是否保留白名单项

### 11.5 设置页

#### 支持内容

- 默认扫描路径
- 是否扫描自定义路径
- 默认保留最近 N 个版本
- 未使用时间阈值
- 是否优先移入回收站
- 白名单管理

## 18. 风险与约束

### 18.1 主要风险

- 误删仍在使用的依赖
- 离线或私服环境无法重新下载
- Gradle 缓存结构复杂，分类难度高
- 文件占用导致删除失败
- 跨平台路径与权限差异

### 18.2 控制策略

- 默认以“建议 + 确认”为主
- 低风险项优先推荐
- 支持白名单
- 支持预演
- 提供风险说明
- 删除失败时清晰反馈
