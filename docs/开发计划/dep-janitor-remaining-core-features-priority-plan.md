## 1. 当前判断

当前项目已经具备以下能力：

- 本地路径发现
- Maven / Gradle 缓存扫描
- 候选项识别首轮
- 白名单与项目引用保护基础版
- 规则编辑与持久化
- 预演与删除执行首轮
- 回收站优先删除策略
- 结果反馈基础版
- 跨页面统一执行方案状态
- Artifact Atlas 按清理优先级排序
- 时间依据分层（最近访问 / 最近修改 / 回退推断）首轮
- 多自定义扫描路径管理（新增 / 启停 / 持久化）首轮

也就是说，项目已经不是“原型”，而是一个 **可运行、可扫描、可执行清理的桌面工具首版**。

但如果目标是达到 **PRD 语义下更完整、可发布、可长期维护的 V1 正式版**，仍有一批关键能力需要继续补齐。

## 3. P1-必须继续做

这些功能建议优先继续推进，属于当前项目最重要的剩余核心能力。

---

### 已于本轮完成

- **P1-M1：全局执行方案状态管理**
  - 已支持在 `Artifact Atlas`、`Cleanup Recipes`、`Simulation` 之间共享同一份执行方案
  - 已支持跨页面加入 / 移出方案
  - 顶部状态区已展示当前方案数量与预计释放空间
- **P1-M2：按清理优先级排序**
  - `Artifact Atlas` 已新增“按清理优先级”排序
  - 排序已综合默认建议、风险等级、候选数量与可释放空间
- **P1-M3：更准确的时间语义（首轮）**
  - 扫描阶段已优先尝试使用 `last access time`
  - 当访问时间不可用时，已显式标记为“最近修改时间（回退推断）”
  - UI 与候选理由已展示时间依据与回退状态
- **P1-M5：多自定义扫描路径支持（首轮）**
  - 已支持新增多条 Maven / Gradle / Wrapper 自定义扫描路径
  - 已支持路径启用/停用与本地持久化
  - 已参与统一扫描流程

---

### P1-M3.2：更准确的时间语义（第二轮增强）

**现状**:
- 当前已具备：
  - `TimeBasis`
  - 最近访问时间优先
  - 最近修改时间回退标识
- 但仍未引入更高可信度的“引用 / 使用痕迹”语义

**为什么重要**:
- “180 天未使用”是 PRD 的关键判断条件之一
- 仅依赖文件系统时间仍会影响候选识别精度

**建议实现内容**:
- 为候选识别引入更准确的时间判定策略：
  - 优先使用可获得的访问/引用痕迹
  - 结合文件系统访问时间与目录活动时间
  - 不可得时再回退为最后修改时间
- 在模型层保留：
  - 时间值
  - 时间依据
  - 是否为回退推断

**验收标准**:
- 候选理由中明确时间依据
- 模型可区分“真实可用时间依据”和“回退依据”
- 风险判断比当前更合理

**优先级**: 高

---

### P1-M4：更完整的失败残留 / 异常缓存识别

**现状**:
- 已识别：
  - `.lastUpdated`
  - `.part`
  - `.tmp`
  - `.part.lock`
  - 部分 Gradle transforms/build-cache
- 但覆盖仍不完整

**为什么重要**:
- 这是最适合默认推荐清理的一类“低风险高收益”候选
- 扩展这部分识别，可以直接提升产品的实用性和安全感

**建议实现内容**:
- 继续扩展 Maven / Gradle 的：
  - 临时文件
  - 失败下载残留
  - 异常缓存目录
  - 已知低风险无效缓存模式
- 增加更多测试样本

**验收标准**:
- 低风险候选覆盖面明显扩大
- 默认建议质量提升
- 不误伤正常缓存结构

**优先级**: 高

---

### P1-M5.2：多自定义扫描路径支持（第二轮增强）

**现状**:
- 当前已支持：
  - 默认路径 override
  - 多条新增扫描路径
  - 启用 / 停用
- 但还缺少更完整的路径管理体验

**为什么重要**:
- PRD 明确提到“手动修改或新增扫描路径”
- 实际开发环境中，用户可能有：
  - 多个 Maven 仓库
  - 自定义 Gradle cache 目录
  - 外接磁盘缓存目录

**建议实现内容**:
- 增加路径编辑能力
- 增加路径有效性校验与错误提示
- 展示路径来源、启用状态、存在状态
- 支持一键打开路径 / 复制路径

**验收标准**:
- 用户可更方便地维护多条扫描路径
- 路径异常有明确提示
- 多条路径参与同一次扫描且状态可见

**优先级**: 中高
