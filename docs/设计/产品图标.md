# Dep Janitor 产品图标重设计说明（V2）

## 目标

为开源桌面客户端 **Dep Janitor** 重新设计产品图标。

该产品面向 Java 开发者，用于**安全扫描并清理本地 Maven / Gradle 依赖缓存**。图标应体现：

- dependency governance（依赖治理）
- visual analysis（结构分析 / 扫描感）
- safety-first control（安全、克制、可控）
- professional desktop developer tool（专业桌面开发工具）

不要使用以下陈词滥调：

- 垃圾桶
- 扫帚
- 刷子
- 通用“系统清理软件”隐喻

---

## 对当前图标的问题判断

当前版本虽然具备“精密装置 + 核心结构”的方向感，但**不适合作为桌面应用图标**，主要问题如下：

1. **细节过多**
   - 多层机械环、刻线、金属槽口太密。
   - 在 Dock、任务栏、启动器等小尺寸场景中会糊成一团。

2. **主体不够突出**
   - 中间紫色核心占比偏小。
   - 用户在 16px / 32px 下无法第一眼记住“核心识别物”。

3. **外圈和边框过重**
   - 外层圆角方形 + 多层圆环都很抢视觉。
   - 反而削弱了真正的品牌主体。

4. **偏游戏道具 / 科幻装置**
   - 观感更像“能量核心”或“传送门”。
   - 不够像开发者桌面工具。

5. **与系统图标语言不协调**
   - 和常见桌面应用并排时显得过暗、过复杂、过插画化。
   - 缺乏专业软件图标常见的清晰轮廓与强主体。

---

## 重设计方向

采用方向：**Precision Core**

一句话定义：

> 一个更大、更清晰的紫色 dependency core，置于深色桌面应用图标底座中央，外围仅保留两段克制而精确的结构化轨道，分别表示 Maven 与 Gradle。

这不是完全推翻原方向，而是：

- 保留“artifact core / dependency core”品牌语言
- 保留 Maven 与 Gradle 的双源语义
- 保留 obsidian lab 的高级感
- 大幅减少机械装饰
- 改为“小尺寸优先”的桌面应用图标表达

---

## 核心视觉结构

### 1. 容器

- 使用 **macOS / Windows 兼容的圆角方形图标容器**
- 深色底：obsidian / graphite
- 容器边缘只允许有**轻微金属感或内发光层次**
- 不要做厚重双边框

### 2. 中央主体

- 中心主体必须是一个**明确、集中的紫色 artifact core**
- 形态可为：
  - 几何晶体
  - engineered node
  - 精密多面体核心
- 核心要比旧版明显更大，成为第一视觉焦点
- 紫色发光必须**克制、内敛、偏技术感**

### 3. Maven / Gradle 双源表达

- 外围只保留两段主结构弧或轨道
- **铜棕色**表示 Maven
- **绿锈青色**表示 Gradle
- 这两种颜色必须是“结构语义”，而不是纯装饰
- 两段弧线需具备：
  - 几何秩序感
  - 工程感
  - 可读性

### 4. 层次数量

- 总体建议控制在 **3 层以内**
  1. 圆角方形底座
  2. 中间结构环 / 双源轨道
  3. 中央紫色核心

避免旧版那种 5~6 层机械嵌套。

## 色彩要求

### 主色

- **Iris Purple**：用于核心高亮与品牌识别

### 辅色

- **Copper Brown**：Maven
- **Verdigris Teal**：Gradle

### 基底

- **Obsidian / Graphite**：深色底板

### 原则

- 紫色是视觉记忆点
- 铜色与青绿色必须少量、精准、结构化出现
- 整体颜色要像专业桌面软件，而不是游戏 UI

---

## 关键约束

### 必须做到

- 在 **16px / 32px / 64px** 下仍有清晰识别度
- 中心主体一眼可见
- 轮廓明确，避免一缩小就脏
- 图标与常见桌面应用并排时不突兀
- 具备“专业开发工具”气质

### 必须避免

- 复杂机甲风刻线
- 过多装饰槽口
- 过度写实金属纹理
- 游戏道具感
- 能量传送门感
- 清理工具刻板印象
- 扁平网页 favicon 感

---

## 设计策略

### 策略 1：主体放大

中央核心相对旧版建议放大 **20% ~ 30%**。

### 策略 2：删除无效细节

移除所有对小尺寸无帮助的：

- 装饰性刻槽
- 重复环形层
- 过细金属切线
- 纯炫技纹理

### 策略 3：保留品牌语义但降低“科幻游戏感”

图标要像：

- 工程分析工具
- 桌面开发工具
- 依赖治理中枢

而不是：

- RPG 道具
- 魔法核心
- 科幻传送器

### 策略 4：优先为桌面图标而设计

先保证 32px 可读，再考虑 1024px 精修。

## 推荐成品观感

理想状态下，图标应给人这样的感觉：

> 一个深色、沉稳、精密的桌面工具图标。中心是发出柔和紫色光的 dependency core，外部以两段简洁而有秩序的铜色与青绿色轨道包围，整体克制、高级、工程化，在小尺寸下也能一眼认出。

## 图像生成 / 设计执行 Prompt（推荐主 Prompt）

```text
Design a premium desktop app icon for “Dep Janitor”, an open-source developer tool for safely scanning and cleaning local Maven and Gradle dependency caches.

Use a macOS and Windows friendly rounded-square icon container with a dark obsidian / graphite base.

At the center, place a larger and clearer purple dependency core / artifact core — a refined engineered crystal or geometric node with restrained iris-purple glow. The core must be the primary focal point and remain recognizable at small sizes.

Around the core, keep only two precise structural arcs / rails to represent the two dependency sources:
- Maven in copper brown
- Gradle in verdigris teal

The icon should feel like a professional desktop developer tool, not a game item, not a sci-fi portal, and not a generic system cleaner.

Visual style: precision instrument, artifact governance, safe cleanup, geometric precision, restrained futuristic, premium desktop software, sophisticated, centered composition, strong silhouette, excellent readability at 16px / 32px / 64px.

Reduce mechanical details. Remove excessive grooves, decorative ring layers, dense panel lines, and heavy ornamental borders.

Use a dark obsidian-lab visual language with subtle material richness, restrained metallic accents, and a controlled purple glow. Keep the composition calm, structured, and engineering-oriented.
```

## 输出要求

最终产物建议包括：

- 1024 × 1024 PNG 母版
- 透明背景版本（如设计工具支持）
- 小尺寸预览图：
  - 64 × 64
  - 32 × 32
  - 16 × 16

在导出进项目时，母版文件放到：

- `app-desktop/src/main/resources/icons/source/dep-janitor-1024.png`

然后执行：

```bash
python3 scripts/generate_app_icons.py
```

生成：

- runtime PNG
- Windows ICO
- macOS ICNS

## 验收标准

满足以下条件才算通过：

- 与系统中其他桌面应用图标并排时不显突兀
- 在 32px 下仍能看出：
  - 深色底座
  - 中央紫色核心
  - 两个来源色彩结构
- 观感更像开发工具，而不是游戏道具
- 与 Dep Janitor 的产品语义一致：依赖治理、扫描分析、安全清理、可控执行
